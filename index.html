À propos de moi

Je suis étudiant en BTS Services Informatiques aux Organisations (SIO) – option SISR (Solutions d'Infrastructure, Systèmes et Réseaux), promotion 2024-2026.

PROJET PRINCIPAL

📌 Infrastructure Docker avec reverse proxy Caddy

Conception et déploiement de sites web isolés avec Docker, Alpine Linux et Caddy

🎯 Objectifs :

  • Créer une infrastructure modulaire pour héberger plusieurs sites web
  • Isoler chaque site dans son propre conteneur Docker
  • Mettre en place un reverse proxy Caddy pour gérer le HTTPS automatiquement
  • Optimiser les images avec Alpine Linux (ultra-légères ~5-15MB)

🏗️ Architecture mise en place :

Site 1 Alpine + PHP Caddy
Site 2 Alpine + PHP Caddy
Site 3 Alpine + PHP Caddy

🛠️ Réalisations techniques :

  • Création de 3 sites web : Site vitrine (HTML/CSS/JS), Blog (PHP/MySQL), Portfolio (HTML/CSS/JS)
  • Dockerfile Alpine : Optimisation multi-stage pour des images ~15MB
  • Configuration Caddy : Reverse proxy avec HTTPS automatique (Let's Encrypt)
  • docker-compose.yml : Orchestration des 3 conteneurs + Caddy
  • Sécurisation : Utilisateur non-root, réseaux internes, volumes isolés
  • Déploiement : Mise en production sur serveur dédié

📄 Exemple de configuration :

docker-compose.yml 
services:
  site1:
    build: ./site1
    networks: [webnet]
  site2:
    build: ./site2
    networks: [webnet]
  caddy:
    image: caddy:alpine
    ports: [80:80, 443:443]
    volumes: [./Caddyfile:/etc/caddy/Caddyfile]
Caddyfile 
site1.bibros.local { reverse_proxy site1:9000 }
site2.bibros.local { reverse_proxy site2:9000 }
site3.bibros.local { reverse_proxy site3:9000 }

🔧 Technologies utilisées :

Docker Alpine Caddy PHP HTML5 CSS3 JavaScript MySQL

📊 Résultats obtenus :

  • 3 sites web déployés et accessibles en HTTPS
  • Images Docker optimisées (15MB au lieu de 100MB+)
  • Infrastructure reproductible avec docker-compose
  • Isolation totale entre les sites (sécurité renforcée)
  • HTTPS automatique géré par Caddy
